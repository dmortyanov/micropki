# Sprint 6 — Статус выполнения требований

**Цель спринта:** Реализовать движок валидации путей сертификатов и набор клиентских инструментов для генерации CSR, запроса сертификатов через API, полной проверки цепочки и статуса отзыва (CRL + OCSP) с логикой автоматического отката.

---

## 1. Структура проекта и клиентские инструменты

| ID    | Требование                                                                     | Приоритет | Статус | Реализация                                              |
|-------|--------------------------------------------------------------------------------|-----------|--------|---------------------------------------------------------|
| **STR-19** | Добавление модулей для валидации и клиентской части. | **Must** | Done | Добавлены `micropki/client.py` и `micropki/revocation_check.py`. Подкоманда `client` интегрирована в основной CLI. |
| **CLI-25** | Подкоманда `client gen-csr` для генерации ключа и CSR. | **Must** | Done | Реализована в `client.py`. Поддерживает RSA/ECC и SAN. Сохраняет ключ с правами 0o600. |
| **CLI-26** | Подкоманда `client request-cert` для запроса через API. | **Must** | Done | Отправляет CSR на сервер репозитория и сохраняет полученный PEM-сертификат. |
| **CLI-27** | Подкоманда `client validate` для проверки всей цепочки. | **Must** | Done | Выполняет построение пути до доверенного корня и проверку подписей/сроков. |
| **CLI-28** | Подкоманда `client check-status` (OCSP + CRL fallback). | **Must** | Done | Реализует логику: сначала OCSP, при неудаче (ошибка/unknown) — проверка через CRL. |

## 2. Валидация путей (RFC 5280)

| ID    | Требование                                                                     | Приоритет | Статус | Реализация                                              |
|-------|--------------------------------------------------------------------------------|-----------|--------|---------------------------------------------------------|
| **VAL-1** | Построение цепочки от листа до доверенного корня. | **Must** | Done | Реализовано в `chain.py`. Корректно упорядочивает сертификаты и находит путь. |
| **VAL-2** | Базовые проверки (подпись, сроки, Basic Constraints). | **Must** | Done | Проверяются подписи всеми звеньями, сроки действия и ограничения CA/PathLength. |
| **VAL-5** | Структурированный результат валидации. | **Must** | Done | Клиент выводит детальный лог проверок — от проверки подписи каждого звена до статуса отзыва. |

## 3. Проверка отзыва (CRL и OCSP)

| ID     | Требование                                                                    | Приоритет | Статус | Реализация                                              |
|--------|-------------------------------------------------------------------------------|-----------|--------|---------------------------------------------------------|
| **REV-1** | Поддержка проверки через CRL (локальный или URL). | **Must** | Done | `revocation_check.py`: загружает CRL, проверяет подпись и наличие серийного номера. |
| **REV-2** | Поддержка проверки через OCSP. | **Must** | Done | `revocation_check.py`: формирует OCSP-запрос, отправляет и проверяет ответ (включая nonce). |
| **REV-3** | Логика предпочтения: OCSP сначала, затем CRL. | **Must** | Done | Реализовано: если OCSP недоступен или вернул Unknown, выполняется попытка через CRL. |
| **REV-4** | Парсинг AIA для поиска OCSPResponder. | **Must** | Done | Автоматически извлекает URL из расширения Authority Information Access сертификата. |

## 4. Расширение репозитория для приема CSR

| ID     | Требование                                                                    | Приоритет | Статус | Реализация                                              |
|--------|-------------------------------------------------------------------------------|-----------|--------|---------------------------------------------------------|
| **REPO-15**| Эндпоинт `POST /request-cert`. | **Must** | Done | `repository.py`: принимает CSR, вызывает логику выпуска CA и возвращает сертификат. |
| **CA-1**   | Поддержка CSR в логике выпуска сертификатов. | **Must** | Done | `ca.py`: извлекает публичный ключ, Subject и SAN из CSR, проверяет подпись CSR перед выпуском. |

**Итого:** В Спринте 6 реализован полноценный клиентский функционал. PKI теперь поддерживает не только выпуск "внутри" CLI, но и взаимодействие по сети через CSR, а также надежную проверку статуса сертификатов согласно RFC 5280.
