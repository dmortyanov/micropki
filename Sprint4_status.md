# Sprint 4 — Статус выполнения требований

**Цель спринта:** Реализовать систему списка отозванных сертификатов (CRL), включая механизм отзыва, автоматическую генерацию CRL, расширение HTTP-репозитория и обновление CLI.

---

## 1. База данных: Схема и логика

| ID    | Требование                                                                     | Приоритет | Статус | Реализация                                              |
|-------|--------------------------------------------------------------------------------|-----------|--------|---------------------------------------------------------|
| **DB-5** | Обновление таблицы `crl_metadata` для отслеживания номеров CRL. | **Must** | Done | В `database.py` добавлена таблица `crl_metadata` (поля `ca_name`, `crl_number`, `last_generated_at`, `path`). |
| **DB-6** | Реализация миграции на версию 2 схемы БД. | **Must** | Done | `database.py`: используется `PRAGMA user_version = 2` для безопасного добавления таблицы. |
| **DB-7** | Функция для обновления статуса сертификата на `revoked`. | **Must** | Done | Метод `CertificateDatabase.update_certificate_status` возвращает false, если сертификат уже был отозван. |

| ID    | Требование                                                                     | Приоритет | Статус | Реализация                                              |
|-------|--------------------------------------------------------------------------------|-----------|--------|---------------------------------------------------------|
