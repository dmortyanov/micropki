# Sprint 7 — Статус выполнения требований

**Цель спринта:** Усилить безопасность PKI за счет внедрения защищенного журнала аудита, механизмов контроля доступа (Rate Limiting), имитации Certificate Transparency и строгого соблюдения политик выпуска (ключи, сроки, SAN).

---

## 1. Система аудита и целостность логов

| ID    | Требование                                                                     | Приоритет | Статус | Реализация                                              |
|-------|--------------------------------------------------------------------------------|-----------|--------|---------------------------------------------------------|
| **AUD-1** | Структурированный аудит в формате NDJSON. | **Must** | Done | `audit.py`: все события логируются в `pki/audit.log` с метаданными и полем целостности. |
| **AUD-2** | Хеш-цепочка для защиты от подделки (SHA-256). | **Must** | Done | Каждая запись содержит хеш текущей строки и `prev_hash` предыдущей, образуя неизменяемую цепочку. |
| **CLI-32** | Подкоманда `audit verify` для проверки цепочки хешей. | **Must** | Done | Реализована полная перепроверка целостности лога. При любом изменении или удалении записи проверка падает. |
| **AUD-4** | Обязательный аудит критических операций. | **Must** | Done | Инициализация CA, выпуск сертификатов, отзыв и отказы в выпуске по политикам фиксируются в логе. |

## 2. Обеспечение политик безопасности (Policy Enforcement)

| ID    | Требование                                                                     | Приоритет | Статус | Реализация                                              |
|-------|--------------------------------------------------------------------------------|-----------|--------|---------------------------------------------------------|
| **POL-3** | Контроль минимального размера ключей (RSA 2048+, ECC 256+). | **Must** | Done | `policy.py`: система отклоняет выпуск сертификатов с небезопасными ключами. |
| **POL-4** | Ограничение максимальных сроков действия (Root=10л, Leaf=1г). | **Must** | Done | `policy.py`: проверка соответствия сроков выпуска установленным лимитам безопасности. |
| **POL-5** | Ограничения SAN и запрет Wildcard по умолчанию. | **Must** | Done | `policy.py`: жесткая проверка типов имен для разных шаблонов (server/client/code_signing). |

## 3. Операционные контроли безопасности

| ID     | Требование                                                                    | Приоритет | Статус | Реализация                                              |
|--------|-------------------------------------------------------------------------------|-----------|--------|---------------------------------------------------------|
| **CTL-1** | Ограничение частоты запросов (Rate Limiting). | **Should** | Done | `ratelimit.py`: реализован алгоритм Token Bucket для защиты Repo и OCSP серверов. |
| **CLI-34** | Флаги `--rate-limit` и `--rate-burst` для серверов. | **Should** | Done | Параметры передаются при запуске `repo serve` или `ocsp serve`. Возвращается HTTP 429 при превышении. |
| **CTL-2** | Имитация Certificate Transparency (CT Log). | **Must** | Done | `transparency.py`: при каждом выпуске запись добавляется в публичный файл `ct.log` с фингерпринтом. |
| **CTL-3** | Команда `ca compromise` для компрометации ключа. | **Must** | Done | `cli.py`: немедленный отзыв с причиной `keyCompromise` и блокировка повторного использования ключа. |

## 4. Расширение БД для безопасности

| ID     | Требование                                                                    | Приоритет | Статус | Реализация                                              |
|--------|-------------------------------------------------------------------------------|-----------|--------|---------------------------------------------------------|
| **DB-10** | Таблица `compromised_keys` для отслеживания плохих ключей. | **Should** | Done | В БД добавлено хранение хешей публичных ключей, использование которых запрещено. |

**Итого:** Система MicroPKI стала защищенной от операционных ошибок и атак типа DoS. Целостность журнала операций гарантирована криптографически, а политики исключают выпуск сертификатов, не соответствующих стандартам безопасности.
