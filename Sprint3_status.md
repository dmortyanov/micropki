## 3. Основная реализация PKI (Интеграция БД и Серийные номера)

| ID     | Требование                                                                    | Приоритет | Статус | Реализация                                              |
|--------|-------------------------------------------------------------------------------|-----------|--------|---------------------------------------------------------|
| **PKI-13** | **Генератор уникальных серийных номеров:** Гарантирует уникальность для всех сертификатов. Рекомендуемый подход: 64-битное составное значение (старшие 32 бита = timestamp, младшие 32 бита = CSPRNG). | **Must** | Done | `serial.py::SerialNumberGenerator.generate()` — комбинирует `time.time()` и `os.urandom(4)`. База данных обеспечивает уникальность через constraint. |
| **PKI-14** | **Автоматическая вставка в БД при выпуске:** Сразу после подписания сертификата и записи на диск, вставляется запись в таблицу `certificates`. | **Must** | Done | Реализовано в `ca.py` во всех трёх функциях выпуска. Данные: serial_hex, subject, issuer, notBefore/After, PEM, status='valid'. |
| **PKI-15** | **Определение схемы БД:** SQLite-база содержит таблицу `certificates` с указанными полями и `UNIQUE` constraint на `serial_hex`. | **Must** | Done | `database.py::CertificateDatabase.init_schema()` создаёт таблицу с нужными полями и `UNIQUE` constraint на `serial_hex`. |
| **PKI-16** | **Миграция БД:** Поддержка простого автоматического механизма миграции. | **Should** | Done | `database.py`: добавлена минимальная миграция через `PRAGMA user_version` и `ALTER TABLE` для отсутствующих колонок |
| **PKI-17** | **Обработка ошибок:** Если вставка в БД не удалась (дубликат серийного номера, нехватка места), выпуск сертификата должен завершиться с ошибкой, файл не должен быть записан, ошибка должна быть залогирована. | **Must** | Done | `ca.py`: Вставка в БД выполняется в транзакции (через `with conn`). При `IntegrityError` выбрасывается `ValueError`, и весь процесс выпуска прерывается. |
